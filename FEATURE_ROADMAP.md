# Bakery App 功能路线图

## 概述

本文档记录 Bakery App 未来计划开发的功能，按优先级和模块组织。

---

## 待开发功能

### 优先级：高

#### 1. 生产环境监控（温度/湿度）
- **描述**：监控后厨环境、醒发箱、冰箱等设备的温度和湿度
- **需求细节**：
  - 设备管理：注册设备，设置名称、位置、类型
  - 实时数据：当前温度、湿度、电池状态
  - 历史记录：按时间序列存储数据
  - 超温报警：App 内推送通知
- **数据记录频率**（按设备优先级）：
  | 设备类型 | 记录频率 |
  |---------|---------|
  | 醒发箱 | 每 1 分钟 |
  | 冰箱 | 每 5 分钟 |
  | 后厨环境 | 每 30 分钟 |
- **仪表盘显示**：
  - 每个设备的当前温度/湿度
  - 状态指示（正常/报警）
  - 历史数据折线图
  - 设备列表（按位置筛选）
- **硬件方案**：
  - 传感器芯片：nRF52810 / nRF52832（BLE，低功耗）
  - 温湿度传感器：SHT40
  - 电池：CR2032 / CR2477
  - 通信：BLE Advertise（低功耗广播）
  - 广播频率：每5分钟一次
  - 距离：10-20米
  - 网关：N1 + BLE USB Dongle + Docker
  - 续航：CR2032 约 3-5 年，CR2477 约 1-2 年
- **BLE Gateway (Docker)**：
  - Python 3.11 + Bleak 库
  - 设备接入数量：5-50个
  - 调用 POST /api/temperature-logs 存库
  - 超温检查写入 alerts 表
- **数据流程**：
  ```
  nRF52 + SHT40 → BLE Advertise → N1 BLE Gateway → POST /api/temperature-logs → DB
                                              ↓
                                        超温检查 → alerts 表
  ```
- **与其他模块关系**：
  - 与打面记录**完全独立**
  - 不替代打面记录中的温度字段
- **状态**：前端原型已完成，BLE Gateway 待创建

#### 2. 工作时钟
- **描述**：实时显示当前进行的工作和下一个工作，支持按员工分配不同工作流程
- **位置**：单独 Tab
- **显示方式**：甘特图 + 红线指示当前时间
  ```
  ┌─────────────────────────────┐
  │ 09:45:23                   │  ← 当前时间（实时）
  │                             │
  │ 进行中                      │
  │ 打面 - 越南法棍            │  ← 当前工作
  │ ████████░░░░  60%         │  ← 进度条
  │ 剩余 12 分钟               │
  │                             │
  │ 下一个                      │
  │ 10:00 整形 - 越南法棍      │  ← 下一个工作
  │ 15 分钟后                  │
  └─────────────────────────────┘
  ```
- **时间体系**：
  - 班次边界固定：早班 06:00-14:00 / 中班 14:00-22:00 / 夜班 22:00-06:00
  - 5分钟 = 最小时间格
  - 1天 = 288 slots (0-287)
- **设计原则**：
  - 数据库只存储原始数据（slot_index, duration_slots）
  - 前端根据当前时间计算红线位置和当前任务
  - 后端只做 CRUD，不做业务计算
- **员工 ↔ 班次关系**：暂不开发，人工排班
- **数据库设计**：
  ```
  shifts 表
  ├── id
  ├── name (早班/中班/夜班)
  ├── start_slot
  ├── end_slot
  ├── color
  └── is_active

  products 表
  ├── id
  ├── name (产品名称)
  ├── dough_id (关联 doughs.id)
  ├── other_ingredients (JSON)
  ├── description
  └── created_at / updated_at

  work_tasks 表
  ├── id
  ├── user_id
  ├── shift_id
  ├── task_type (打面/整形/醒发/准备馅料/烘烤/其他)
  ├── material_id (关联 materials.id)
  ├── slot_index (0-287)
  ├── duration_slots
  ├── work_date
  ├── notes
  └── created_at
  ```
- **slot 换算**：`slot_index = 小时 × 12 + floor(分钟/5)`
  - 06:00 = 72, 14:00 = 168, 22:00 = 264
- **状态**：待开发（前端原型已完成，数据库表和后端 API 待创建）

#### 3. 通用记录系统
- **描述**：通用记录页面，支持预设模版，适合各类运营记录
- **需求细节**：
  - 记录中心页面：选择模版 → 填写表单 → 提交 → 历史记录查看
  - 支持多种记录模版（见下方）
- **权限**：仅 admin 可访问
- **位置**：放在「更多」Tab 内
- **数据库设计**：
  ```
  records 表
  ├── id
  ├── template_type (设备维护/采购/报损/清洁/食安)
  ├── record_date
  ├── operator (操作人)
  ├── data (JSON 格式存储模版特定字段)
  ├── created_at
  └── updated_at
  ```
- **记录模版类型**：
  1. **设备维护记录** - 设备名称、维护类型（保养/维修/配件）、日期、操作人、备注
  2. **原料采购记录** - 采购日期、供应商、原料名称、数量、单价、总价、备注
  3. **报损记录** - 日期、原料/产品名称、数量、原因（过期/损坏/其他）、操作人
  4. **清洁消毒记录** - 日期、区域（后厨/冷区/设备）、清洁方式、操作人、备注
  5. **食品安全检查** - 检查日期、检查项目（温度/卫生/效期）、结果（合格/不合格）、整改措施、操作人
- **状态**：待开发

#### 4. 库存管理
- **描述**：追踪面粉、黄油、糖等原材料的库存和消耗
- **需求细节**：
  - 记录每次采购的原材料数量和价格
  - 打面记录自动扣减对应原材料库存
  - 库存低于阈值时提醒
- **状态**：待开发

#### 5. 成本计算
- **描述**：根据配方原料配比和原材料价格自动计算每个配方的成本
- **需求细节**：
  - 配方成本 = Σ(原料用量 × 原料单价)
  - 支持按批次计算总成本
  - 显示利润率（如果有售价）
- **状态**：待开发

#### 11. 原材料删除保护（软拦截方案）

- **描述**：删除原材料时检查是否被配方引用，给予提醒
- **需求细节**：
  1. **后端软拦截**：删除前查询 `dough_ingredients_current` 和 `preparation_ingredients_current` 是否有引用，如有返回被哪些配方使用，但仍允许删除
  2. **前端确认弹窗**：显示引用列表，用户确认后强制删除
  3. **配方页孤配料提醒**：配方详情页检测 ingredients 中 material_id 是否仍存在于 materials 表，如不存在则标记为「已删除的原材料」
- **待决定**：是否去掉 `preparation_ingredients_current` 的 FK 约束（`fk_prep_mat`）改为纯代码软约束
- **状态**：待开发

## 已完成功能

### 配方管理（doughs）
- 创建/编辑/删除配方
- 原料配比（预发酵、主面团、后加阶段）
- 版本历史管理，可回滚
- 预期出缸温度设置
- 拖拽排序原料顺序
- **数据库表**：
  - `doughs` - 配方主表
  - `dough_versions` - 版本记录
  - `dough_ingredients_current` - 当前版本原料
  - `dough_ingredients_archive` - 历史版本原料

### 半成品管理（preparations）
- 创建/编辑/删除半成品
- 原料配比
- 版本历史管理，可回滚
- 拖拽排序原料顺序
- **数据库表**：
  - `preparations` - 半成品主表
  - `preparation_versions` - 版本记录
  - `preparation_ingredients_current` - 当前版本原料
  - `preparation_ingredients_archive` - 历史版本原料

### 打面记录
- 批次管理（自动生成批次号）
- 温度状态颜色提示
- 按日期分组显示
- 打面记录详情查看

### 原材料管理（materials + ingredients）
- 原材料 CRUD
- 分类：面粉、油脂、糖、盐等
- 制造商、规格、价格信息

### 用户权限
- 管理员/普通用户角色
- 登录认证

### 前端优化
- 配方拖拽排序（@dnd-kit）
- 统一编码表（materials）作为核心
- 配方入口移至「更多」菜单（需 canViewRecipes 权限）
- 局部缓存服务（LocalStorage + 5分钟过期）

### 配方损耗率
- 原料级别损耗率（loss_rate）字段
- 新建/编辑配方时可设置每个原料的损耗率
- 默认值 1%，适用于烘焙制作过程中的正常损耗

### 打面计算器
- 选择配方或半成品
- 输入目标成品重量
- 自动计算各原料实际用量（含损耗）
- 适用于面团、馅料、半成品等多种制作品
